Queries

greeting

Description

Returns a greeting message with the provided name

Response

Returns a String!

Arguments
Name Description
name - String! Default = "Pavel"

Example

Query
query Greeting($name: String!) {
  greeting(name: $name)
}
Variables
{"name": "Pavel"}
Response
{"data": {"greeting": "xyz789"}}
Queries

projects

Description

Returns all projects in the system

Response

Returns [ProjectDTO!]!

Example

Query
query Projects {
  projects {
    id
    customer {
      id
      name
      emailAddress
    }
    translator {
      id
      name
      emailAddress
    }
    targetLanguage
    sourceFile
    translatedFile
    state
    createdAt
  }
}

{
  "data": {
    "projects": [

        "translatedFile": "abc123",
        "state": "CREATED",
        "createdAt": "2007-12-03T10:15:30Z"
      }
    ]
  }
}

Types

ID

Description

The ID scalar type represents a unique identifier, often used to refetch an object or as key for a cache. The ID type appears in a JSON response as a String; however, it is not intended to be human-readable. When expected as an input type, any string (such as "4") or integer (such as 4) input value will be accepted as an ID.

Example
"4"
Types

ProjectDTO

Description

Represents a translation project in the system

Fields
Field Name Description
id - ID! Unique identifier of the project
customer - UserDTO! Customer who requested the translation
translator - UserDTO Translator assigned to the project (optional)
targetLanguage - String! Target language for the translation (language tag format)
sourceFile - String! Base64 encoded source file content
translatedFile - String Base64 encoded translated file content (available after completion)
state - ProjectStateDTO! Current state of the project
createdAt - DateTime! When the project was created
Example
{
  "id": "4",
  "customer": UserDTO,
  "translator": UserDTO,
  "targetLanguage": "xyz789",
  "sourceFile": "xyz789",
  "translatedFile": "xyz789",
  "state": "CREATED",
  "createdAt": "2007-12-03T10:15:30Z"
}
Types

ProjectStateDTO

Description

Possible states of a translation project

Values
Enum Value Description

CREATED

 Project has been created but not yet assigned

ASSIGNED

 Project has been assigned to a translator

COMPLETED

 Translation work has been completed

APPROVED

 Project has been approved by the customer

CLOSED

 Project is closed and archived
Example
"CREATED"

Description

The String scalar type represents textual data, represented as UTF-8 character sequences. The String type is most often used by GraphQL to represent free-form human-readable text.
